[ts-00023] When parsing structured data (like notes, messages, lists) and initial parsing fails, re-examine the raw format closely to identify the exact pattern. Adjust parsing logic to match the observed structure—for example, if entries are separated by double newlines, split by delimiter rather than iterating lines with state tracking. Test parsing on a subset of data before applying to the entire dataset.
[ts-00024] When parsing structured content with clear delimiters between entries (e.g., double newlines in notes), use delimiter-based splitting instead of line-by-line state tracking. This approach is more reliable and simpler for patterns where entries are separated by consistent delimiters.
[ts-00405] When parsing Simple Note content with repeating entries separated by blank lines and line-by-line state tracking parsing fails (returns empty list), immediately switch to delimiter-based splitting using content.split('\n\n') to isolate each entry block, then parse each block separately. This approach is simpler and more reliable for patterns where entries are clearly separated by consistent delimiters. Never fall back to manual extraction; instead, debug the parsing algorithm by testing on a small subset and adjusting the parsing strategy.
[ts-00475] When parsing structured content yields empty results despite visible matches in raw content, immediately debug the parsing algorithm: (1) Test the parsing on a small subset of the raw content (e.g., first few lines or first entry). (2) Compare extracted data with manual inspection to identify the exact pattern. (3) Adjust the parsing strategy: if using line-by-line state tracking, switch to delimiter-based splitting (e.g., content.split('\n\n')) or regex patterns matching the observed structure. (4) Verify the corrected algorithm extracts the expected data before processing the entire dataset. This ensures scalability and correctness for similar future tasks, avoiding manual extraction.
